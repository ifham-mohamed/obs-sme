## Purpose

`research/` is the analytical surface — Jupyter notebooks that produce the F1–F6 thesis findings, the labelled dataset that the classifier trains on, the figures the thesis ships. Unlike `ml/` (production code), this is the *researcher's surface*: it reads from production replicas, doesn't write back. Lives outside `backend/` + `ml/` because the lifecycle is different — notebooks are exploratory; they don't need to deploy.

## How to start building

`research/` builds *behind* `ml/` + `backend/` — the notebooks need real data from the production replicas, and the labelling data comes out of Phase 3 of the roadmap.

1. **Set up the notebook environment first.** `requirements-research.txt` with `jupyterlab`, `pandas`, `scipy`, `matplotlib`, `psycopg2-binary`, `pyarrow`. A `.env.research` file pointing to the read-only Postgres replica + the ONNX model URL.
2. **Calibration set first (Phase 3a).** Hand-pick the 20 calibration docs with the domain expert; commit as `data/calibration_set_v1.csv`. The annotator calibration test depends on this file existing.
3. **Pre-pilot data — already done.** The 40-respondent Google Forms scan is in `data/prepilot_2025-09.csv` (per [01_M1_1](01_M1_1_Research_Motivation_Evidence.md)). Redact PII before commit + move under `research/data/` (currently elsewhere if at all).
4. **Labelling batches (Phase 3b–c).** Each batch CSV lives at `data/labeling/batch_NN.csv`. The sampler in `ml/m1/data/samplers.py` emits these; Label Studio re-imports the annotated version. Versioned, append-only — never overwrite a batch.
5. **`test_split.parquet`.** Once the full 800-label set exists, run `scripts/build_test_split.py` (lives in `ml/m1/data/`) to produce + hash-pin the held-out test set. The SHA-256 goes into `model_registry.json`.
6. **Notebook scaffolds (Phase 5).** Start with `findings_classifier_evaluation.ipynb` (Phase 3 outputs feed it). Then `findings_secondary_diffusion.ipynb` (Phase 4 outputs feed it). Then `findings_lag_analysis.ipynb` + `findings_alert_effectiveness.ipynb` (depend on SME survey data, Phase 5).
7. **Figures.** Each notebook writes to `figures/*.png` — commit the PNGs (small) so thesis reviewers can see the chart without re-running.
8. **Pre-registration.** Before unblinding the data, write `research/preregistration.md` listing the hypotheses + tests (the pattern is in [08_M1_1 §Validation](08_M1_1_Research_Findings_Extraction.md)).

The notebooks are the **thesis artifact** — they must re-run end-to-end against the production replica without errors. CI runs `nbconvert --execute` on each to catch drift.

## Dependencies

- **Postgres replica** — every notebook reads from a read-only replica. The notebooks must NOT write back (enforced by the replica's permissions).
- **`storage/models/m1/v*/`** ([15_M1_5_Storage_Folder_Guide.md](15_M1_5_Storage_Folder_Guide.md)) — `findings_classifier_evaluation.ipynb` loads the ONNX model from here.
- **Label Studio** ([09_M1_2_Annotation_Workflow_IAA_Protocol.md](09_M1_2_Annotation_Workflow_IAA_Protocol.md)) — external tool; `data/labeling/*.csv` is the import/export format.
- **`ml/m1/data/samplers.py`** ([15_M1_1_ML_Folder_Guide.md](15_M1_1_ML_Folder_Guide.md)) — produces the batch CSVs.
- **SME survey backend** ([backend tasks/services](15_M1_2_Backend_Folder_Guide.md)) — populates `m1_sme_awareness_responses`, which `findings_lag_analysis.ipynb` reads.

## Tests & acceptance criteria

- **Notebook reproducibility.** Each notebook re-runs end-to-end against the production replica without errors. CI runs `jupyter nbconvert --execute` on every PR that touches `research/notebooks/`.
- **Figures byte-identical.** PNG outputs are deterministic — same data → same figure. CI snapshot test.
- **Statistical tests pre-registered.** Every finding (F1–F6) has a pre-registered test in `research/preregistration.md`; the notebook implements that exact test. Post-hoc deviations get a methodology footnote.
- **Sample-size disclaimers.** Any finding with N < 30 per slice is reported with "low confidence — N=X" in both notebook + thesis.
- **Per-finding DoD.** F1: median portal lag with 95% CI on ≥ 200 regulations. F2: same for news. F3: median SME lag urban vs rural with Mann-Whitney p-value, n ≥ 100. F4: channel ranking with ≥ 10 SMEs per sector. F5: Kruskal-Wallis on 3 language groups, n ≥ 30 each. F6: DiD with parallel-trends check.
